CANDERA Systems Report

Technical Synthesis.

Transitioning Candera from a static prototype to a Headless Ecosystem. Powered by Next.js App Router and Payload CMS for localized, low-latency artisan commerce.

Architectural Strategy

The following strategy outlines the synergistic relationship between high-end editorial storytelling and modern engineering. By leveraging React Server Components, we ensure the "Aesthetic-Usability Effect" is maintained without sacrificing mobile load speeds.

01

The Headless Core

Separating the content layer (Payload) from the presentation (Next.js) allows for multi-channel distribution across Web, Etsy, and future mobile app iterations.

02

Relational Integrity

Utilizing PostgreSQL to map complex relationships between batches, scent notes, and therapeutic properties for the Scent Alchemist engine.

03

Edge Optimization

Images are dynamically served via WebP with stripped EXIF metadata, ensuring that luxury editorial photography doesn't hinder performance.

Accessibility Remediation.

Based on the recent ADA Accessibility Audit, we have prioritized four technical growth areas to achieve WCAG 2.1 AA compliance across the new dashboard and storefront.

ARIA Tabbed Interface

Issue: Standard buttons lacked state communication for screen readers.

Remediated role="tablist" added

Color Contrast Adjustment

Issue: Lavender (#9A91A4) with white text fell below 4.5:1 ratio.

Remediated Switched to Obsidian text

Semantic Heading Hierarchy

Issue: Page skipped H2 levels, confusing screen reader navigation.

Remediated H1 -> H2 structure fixed

Canvas Fallback Descriptions

Issue: Charts were invisible to assistive technology users.

Remediated aria-label descriptions added
